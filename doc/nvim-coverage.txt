================================================================================
INTRODUCTION                                                   *nvim-coverage*

nvim-coverage displays code coverage from lcov files directly in the editor.
It reads a pre-generated lcov file and provides sign column markers, virtual
text hit counts, a branch overlay popup, a summary popup, and quickfix/loclist
navigation. It does not run tests or invoke any external tools itself.

  1. Setup................................................|nvim-coverage-setup|
  2. Commands.........................................|nvim-coverage-commands|
  3. Lua API...........................................|nvim-coverage-lua-api|
  4. Neotest consumers.........................|nvim-coverage-neotest|

SETUP                                                    *nvim-coverage-setup*

                                                            *coverage.setup()*
coverage.setup({opts})
    Initial setup function. Must be called before using the plugin.

    Usage: >

    require('coverage').setup({
      auto_reload = { enabled = true },
    })
<

Valid keys for {opts}:

    file: ~
        Path (string) or list of paths (table) to the lcov file. When a list
        is given the first existing file is used. Used by |:CoverageLoad| and
        |coverage.load()| when no file argument is given.
        Defaults to: >
            { "lcov.info", "cover/lcov.info", "coverage/lcov.info", "target/lcov.info" }
<

    auto_reload: ~
        Table controlling file watching behaviour.

        auto_reload.enabled: ~
            When true, the lcov file is watched for changes after loading.
            Signs are updated automatically whenever the file changes on disk.
            The watcher stops when |:CoverageClear| or |coverage.clear()| is
            called.
            Defaults to: `false`

        auto_reload.timeout_ms: ~
            Debounce delay in milliseconds before reloading after a change.
            Defaults to: `500`

    commands: ~
        When true, register the |nvim-coverage-commands| on setup.
        Defaults to: `true`

    on_load: ~
        A Lua function called after coverage is loaded.
        Example: >

        require('coverage').setup({
            on_load = function()
                vim.notify("Coverage loaded")
            end,
        })
<

    sign_group: ~
        Name of the sign group used when placing signs. See |sign-group|.
        Defaults to: `"coverage"`

                                                         *coverage.highlights*
Valid keys for {opts.highlights}:

    Each highlight group accepts: `fg`, `bg`, `sp`, `style`, `link`.
    See |highlight-gui| and |:highlight-link|.

    covered: ~
        Highlight for covered lines.
        Defaults to: `{ fg = "#B7F071" }`
    uncovered: ~
        Highlight for uncovered lines.
        Defaults to: `{ fg = "#F07178" }`
    partial: ~
        Highlight for partially covered lines (branch coverage).
        Defaults to: `{ fg = "#AA71F0" }`
    report_border: ~
        Border highlight for the summary popup.
        Defaults to: `{ link = "FloatBorder" }`
    report_normal: ~
        Normal text highlight for the summary popup.
        Defaults to: `{ link = "NormalFloat" }`
    report_cursor_line: ~
        Cursor line highlight for the summary popup.
        Defaults to: `{ link = "CursorLine" }`
    report_header: ~
        Header row highlight for the summary popup.
        Defaults to: `{ style = "bold,underline", sp = "fg" }`
    report_pass: ~
        Highlight for files above the min_coverage threshold.
        Defaults to: `{ link = "CoverageCovered" }`
    report_fail: ~
        Highlight for files below the min_coverage threshold.
        Defaults to: `{ link = "CoverageUncovered" }`
    line_hits: ~
        Highlight for virtual text hit counts.
        Defaults to: `{ link = "Comment" }`

                                                              *coverage.signs*
Valid keys for {opts.signs}:

    covered: ~
        Sign for covered lines.
        Defaults to: `{ hl = "CoverageCovered", text = "▎" }`
    uncovered: ~
        Sign for uncovered lines.
        Defaults to: `{ hl = "CoverageUncovered", text = "▎" }`
    partial: ~
        Sign for partially covered lines.
        Defaults to: `{ hl = "CoveragePartial", text = "▎" }`

    Each sign also accepts an optional `priority` key. See |sign-priority|.

                                                             *coverage.report*
Valid keys for {opts.report}:

    width: ~
        Width of the popup window as a fraction of the terminal width.
        Defaults to: `0.70`
    height: ~
        Height of the popup window as a fraction of the terminal height.
        Defaults to: `0.50`
    min_coverage: ~
        Coverage percentage threshold. Files below this value are highlighted
        with `report_fail`, files above with `report_pass`.
        Defaults to: `80.0`
    borders: ~
        Border characters for the popup. See plenary window API.
    window: ~
        Additional window options passed to the popup. See plenary window API.

                                                           *coverage.line_hits*
Valid keys for {opts.line_hits}:

    enabled: ~
        When true, virtual text hit counts are shown automatically after
        |coverage.load()|.
        Defaults to: `false`

    position: ~
        Where to place the virtual text. One of `"eol"`, `"right_align"`,
        or `"inline"`. See |nvim_buf_set_extmark()|.
        Defaults to: `"eol"`

COMMANDS                                              *nvim-coverage-commands*

:CoverageLoad[!] [{file}]                                      *:CoverageLoad*
    Load an lcov file. If {file} is not given, uses the `file` config option.
    Signs are placed immediately if already visible, otherwise cached until
    |:CoverageShow| is called.

    With [!]: open an interactive picker (`vim.ui.select`) over all `*.info`
    files found under the current working directory. If exactly one file is
    found it is loaded directly without prompting. Any {file} argument is
    ignored when [!] is given.

:CoverageShow                                                  *:CoverageShow*
    Show coverage signs. Requires a prior |:CoverageLoad|.

:CoverageHide                                                  *:CoverageHide*
    Hide coverage signs.

:CoverageToggle                                              *:CoverageToggle*
    Toggle coverage sign visibility.

:CoverageClear                                                *:CoverageClear*
    Remove signs, clear the cached report, stop the file watcher, and disable
    virtual text and branch overlay.

:CoverageReport                                            *:CoverageReport*
    Open the coverage summary popup. Requires a prior |:CoverageLoad|.

:CoverageHeatmap                                        *:CoverageHeatmap*
    Open a full-screen text-mode treemap. Files are sized proportionally to
    their line count and colored by coverage % (dark red → dark amber → dark
    green). Press `<CR>` to open the file under the cursor, `q` or `<Esc>` to
    close. Requires a prior |:CoverageLoad|.

:CoverageToggleLineHits                            *:CoverageToggleLineHits*
    Toggle virtual text hit counts on/off. Shows execution count for every
    instrumented line (e.g. `× 42`). Requires a prior |:CoverageLoad|.

:CoverageToggleBranchHits                        *:CoverageToggleBranchHits*
    Toggle branch overlay popup on/off. When enabled, moving the cursor to a
    partially covered line shows a floating window with per-branch execution
    counts. Requires a prior |:CoverageLoad|.

:CoverageQuickfix [{filter}]                              *:CoverageQuickfix*
    Populate the quickfix list with one entry per file showing coverage
    percentage and uncovered/partial line counts, sorted by coverage ascending
    (worst first). Opens the quickfix window.
    {filter} can be `uncovered` to include only files that have uncovered lines.
    Requires a prior |:CoverageLoad|.

:CoverageLoclist [{sign_type}]                              *:CoverageLoclist*
    Populate the location list for the current window with lines of the given
    type in the current buffer, sorted by line number. Opens the location list.
    {sign_type} is `uncovered` (default) or `partial`.
    Requires a prior |:CoverageLoad|.

:CoverageBrowser                                    *:CoverageBrowser*
    Generate an HTML coverage report from the loaded lcov file using `genhtml`
    and open `index.html` in the system browser (`open` on macOS, `xdg-open`
    on Linux). Requires a prior |:CoverageLoad|.
    Only registered when `genhtml` is found on $PATH at startup.

LUA API                                                *nvim-coverage-lua-api*

                                                             *coverage.load()*
coverage.load({file}, {place})
    Load an lcov file. If {file} is nil, uses `config.opts.file`.
    When {place} is true, signs are placed immediately; otherwise signs are
    placed only if already visible, else cached until |coverage.show()|.
    For the interactive file picker use |:CoverageLoad| with [!].
    Also starts the file watcher when `auto_reload.enabled` is true.

                                                            *coverage.show()*
coverage.show()
    Show coverage signs. No-op if coverage has not been loaded or signs are
    already visible.

                                                            *coverage.hide()*
coverage.hide()
    Hide coverage signs.

                                                          *coverage.toggle()*
coverage.toggle()
    Toggle sign visibility.

                                                           *coverage.clear()*
coverage.clear()
    Remove signs, clear the cached report, stop the file watcher, and disable
    virtual text and branch overlay.

                                                         *coverage.report()*
coverage.report()
    Open the coverage summary popup.

    Summary popup keybindings:
        `s`       Sort by coverage ascending
        `S`       Sort by coverage descending
        `H`       Jump to top entry
        `<CR>`    Open file under cursor
        `?`       Toggle help screen
        `q`       Close
        `<Esc>`   Close

                                                      *coverage.heatmap()*
coverage.heatmap()
    Open the full-screen treemap heatmap. Requires a prior |coverage.load()|.

    Heatmap keybindings:
        `<CR>`    Open file under cursor
        `q`       Close
        `<Esc>`   Close

                                               *coverage.toggle_line_hits()*
coverage.toggle_line_hits()
    Toggle virtual text hit counts. Shows execution count for every
    instrumented line as virtual text (e.g. `× 42`). Requires a prior
    |coverage.load()|.

                                             *coverage.toggle_branch_hits()*
coverage.toggle_branch_hits()
    Toggle branch overlay popup. When enabled, moving the cursor to a
    partially covered line shows a floating window with per-branch execution
    counts. Requires a prior |coverage.load()|.

                                                        *coverage.quickfix()*
coverage.quickfix({filter})
    Populate the quickfix list with one entry per file, sorted by coverage
    ascending (worst first). Each entry shows coverage percentage and counts
    of uncovered/partial lines. Opens the quickfix window.
    {filter} is optional — pass `"uncovered"` to skip fully-covered files.
    Requires a prior |coverage.load()|.

                                                         *coverage.loclist()*
coverage.loclist({sign_type})
    Populate the location list for the current window with lines of the given
    type in the current buffer, sorted by line number. Opens the location list.
    {sign_type} is `"uncovered"` (default) or `"partial"`.
    Requires a prior |coverage.load()|.

                                                       *coverage.jump_next()*
coverage.jump_next({sign_type})
    Jump to the next sign of {sign_type} in the current buffer.
    {sign_type} is one of `"covered"`, `"uncovered"`, `"partial"`.
    Defaults to `"covered"`.

                                                       *coverage.jump_prev()*
coverage.jump_prev({sign_type})
    Jump to the previous sign of {sign_type} in the current buffer.
    {sign_type} is one of `"covered"`, `"uncovered"`, `"partial"`.
    Defaults to `"covered"`.

                                                    *coverage.browser()*
coverage.browser()
    Generate an HTML report from the loaded lcov file using `genhtml` and open
    it in the system browser. Requires `genhtml` on $PATH and a prior
    |coverage.load()|.

NEOTEST CONSUMERS                                       *nvim-coverage-neotest*

The plugin ships built-in neotest consumers that reload coverage automatically
after every test run. Require them directly in your neotest setup.

                                                    *coverage.neotest*
require("coverage.neotest")
    Generic consumer. Reloads the configured lcov file after every completed
    test run. Suitable for any language that writes the lcov file during the
    test run (e.g. Rust with cargo-llvm-cov).

    Usage: >
        require("neotest").setup({
          consumers = {
            coverage = require("coverage.neotest"),
          },
        })
<

                                                    *coverage.neotest.go*
require("coverage.neotest.go")
    Go consumer. After tests finish, converts `coverage.out` in the current
    working directory to `coverage/lcov.info` using `go tool cover`, then
    reloads coverage. Expects tests to produce a `coverage.out` file (e.g.
    via `-coverprofile=coverage.out`).

    Usage: >
        require("neotest").setup({
          consumers = {
            coverage_go = require("coverage.neotest.go"),
          },
        })
<
